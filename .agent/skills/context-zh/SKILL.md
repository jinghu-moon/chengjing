# 上下文工程 - 行为指令

这是一个**行为控制** skill，不是知识库。当用户发出指令时，Claude 必须**改变自己的行为**。

---

## 行为指令表

| 指令 | Claude 必须执行的行为 |
|------|----------------------|
| `省token` | 立即切换到**极简模式**，后续响应必须精简 |
| `太长了` | 立即压缩当前响应，删除冗余内容 |
| `写文件` | 所有输出写入文件，不在对话中展示 |
| `正常模式` | 恢复正常响应模式 |

---

## 指令 1: 省token（极简模式）

**触发词**: 省token、节省token、简洁点

**Claude 必须立即执行**:
1. 回复 `✓ 已切换极简模式`
2. 后续所有响应遵循极简规则

**极简规则**:
- 响应长度减少 70%+
- 不使用表格（除非必要）
- 不解释原理，直接给答案
- 代码只给关键部分
- 大输出自动写入文件

**响应格式**:
```
✓ [一句话确认]
[关键信息，不超过3行]
```

---

## 指令 2: 太长了（压缩响应）

**触发词**: 太长了、简短点、压缩

**Claude 必须立即执行**:
1. 回复 `✓ 压缩版：`
2. 将上一条响应压缩为 1-3 句话

---

## 指令 3: 写文件（输出卸载）

**触发词**: 写文件、存文件、卸载

**Claude 必须立即执行**:
1. 将所有输出写入文件
2. 只返回文件路径

**示例**:
```
用户: 分析架构，写文件
Claude: ✓ 已写入 docs/architecture.md
```

---

## 指令 4: 正常模式（恢复）

**触发词**: 正常模式、恢复、详细点

**Claude 必须立即执行**:
1. 回复 `✓ 已恢复正常模式`
2. 恢复正常响应长度和格式
